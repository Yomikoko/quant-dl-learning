# 阶段3 笔记：过拟合诊断

## 诊断1：训练/验证 Loss 曲线

**结论：模型没学到规律，但不是过拟合。**

```
Epoch  1: Train 0.6926  Val 0.6939
Epoch 50: Train 0.6877  Val 0.6940
         ↓                    ↓
      几乎不降              也几乎不降
```

- 两条线都没显著下降 → 模型没学到规律
- 两条线没有明显分叉 → 不存在过拟合
- **这次模型的问题不是过拟合，是"欠学习"——数据里没信号可学**

**过拟合的标志**：
```
Epoch  1: Train 0.650  Val 0.670
Epoch 20: Train 0.320  Val 0.690
           ↑ 一直降      ↑ 触底反弹
                      ← Gap 越来越大 = 过拟合
```

## 诊断2：Walk-Forward 验证

不再是一次切分，而是 5 次滚动验证：

```
Fold 1: 训[2020-2021] 测[2022前半]
Fold 2: 训[2020-2022] 测[2022后半]
Fold 3: 训[2020-2023] 测[2023前半]
...

结果：所有 fold 的超额收益都在 ±4% 内，没有稳定优势
标准差 1.6%，确认了结论
```

**和单次 split 的区别**：
- 单次 split："这次可能刚好运气不好"
- Walk-forward 5 次："每次都运气不好 = 不是运气问题"

## 诊断3：复杂度对比

| 模型 | 参数 | Test Acc | 超额 |
|------|------|----------|------|
| 简单 MLP | 4,897 | 51.9% | -10.8% |
| LSTM (h=16) | 2,769 | 62.7% | +0.0% |
| LSTM (h=64) | 20,289 | 58.4% | -4.3% |
| LSTM (h=128) | 65,857 | 62.7% | +0.0% |

**参数增加 24 倍，没有带来任何系统性提升。** 说明：
- 不是模型不够复杂
- 是输入特征的信息量已经到天花板
- 继续堆模型 = 只是在拟合更多噪声

## 核心收获

1. 过拟合的定义：训练 loss 降、验证 loss 不降或上升
2. 这个模型的问题是"欠学习"不是"过拟合"——但两者导致的实盘表现都是灾难
3. Walk-forward 比一次 split 更可信
4. 模型复杂度不是免费的——增加参数必须带来验证集上的改进，否则就是浪费
5. **真正该做的：不是找更好的模型，是找更好的数据（阶段1的情感信号、资金流、宏观指标）**
